Free users (Groq BYOK) bypass the Worker entirely and call the Groq API directly from their browser. The Worker is only used for paid credit sessions.

Security: why email alone was not enough

The old flow issued a session token to anyone who typed an email address. Anyone who knew a friend's email could use their credits. The new OTP flow requires access to the actual inbox. The 6-digit code expires in 10 minutes, has a 5-attempt lockout, and uses constant-time comparison to prevent timing attacks.

Step 1 — Install and log in CLI
1
Cloudflare Wrangler setup
Install wrangler, log in, and create the KV namespace that stores credit balances.
# Enter the worker directory
cd prepaired/worker
npm install

# Log into your Cloudflare account (opens browser)
npx wrangler login

# Create the KV namespace for the credit ledger
npx wrangler kv:namespace create CREDITS
# → id = "abc123abc123..."  ← copy this

npx wrangler kv:namespace create CREDITS --preview
# → preview_id = "def456def456..."  ← copy this

Open wrangler.toml and replace both placeholder values with the IDs you just copied.

# wrangler.toml — update these two lines
id = "abc123abc123..."
preview_id = "def456def456..."
Step 2 — Set all secrets CLI
2
Six secrets — set once, never committed
Run each command. Wrangler prompts for the value with hidden input. Nothing is stored locally.
SecretWhere to get it
ANTHROPIC_API_KEYconsole.anthropic.com → API Keys → Create Key. Use a dedicated key for PrepAIred.
SESSION_SECRETGenerate locally (command below). 32 random bytes.
ADMIN_SECRETGenerate locally (same method). Used to protect the gift endpoint.

# Generate random secrets (run twice, save outputs)

npx wrangler secret put ANTHROPIC_API_KEY
npx wrangler secret put SESSION_SECRET
npx wrangler secret put ADMIN_SECRET

# Skip WHOP_WEBHOOK_SECRET for now — set it after Step 4
⚠
Keep ADMIN_SECRET safe. Anyone with it can grant unlimited credits to any email. Store it in a password manager, not in Slack or notes.
Step 3 — Verify sending domain in Resend Browser
3
Add DNS record so Resend can send from ijneb.dev
One DNS record in Cloudflare. Takes 2 minutes.
  • Resend Dashboard → Domains → Add Domain → enter ijneb.dev
  • Resend shows you a TXT record to add. Copy it.
  • Cloudflare Dashboard → ijneb.dev → DNS → Add Record → paste the TXT record
  • Back in Resend → click Verify. Usually instant since ijneb.dev is already on Cloudflare.
→
Once verified, OTP emails will come from PrepAIred <prepaired@ijneb.dev>. Free tier covers 3,000 emails/month — more than enough at this scale.

Step 6 — Configure Whop webhook Browser + CLI
6
Wire Whop purchases to the Worker
Whop fires a signed webhook every time someone buys. The Worker receives it and adds credits.

Whop Dashboard → Developer → Webhooks → Add endpoint

https://prepaired-api.YOURNAME.workers.dev/webhook/whop

Subscribe to: payment.succeeded and membership.created

Copy the signing secret Whop shows you, then:

npx wrangler secret put WHOP_WEBHOOK_SECRET
# Paste the signing secret when prompted
Step 7 — Patch index.html CLI
7
Wire Worker URL and Whop checkout links into the frontend
Three sed commands then push. macOS users: see the note below.
# From the repo root
cd ..

# Replace Worker URL placeholder
sed -i "s|https://prepaired-api.ijnebzor.workers.dev|https://prepaired-api.YOURNAME.workers.dev|g" index.html

# Replace Whop checkout URLs
sed -i "s|WHOP_SINGLE_URL|https://whop.com/checkout/plan_YOUR_SINGLE_ID|g" index.html
sed -i "s|WHOP_5PACK_URL|https://whop.com/checkout/plan_YOUR_5PACK_ID|g" index.html
sed -i "s|WHOP_15PACK_URL|https://whop.com/checkout/plan_YOUR_15PACK_ID|g" index.html

# Push to GitHub → GitHub Pages auto-deploys in ~2 minutes
git add index.html
git commit -m "config: wire Worker URL and Whop checkout URLs"
git push
⚠
macOS: sed -i requires an empty string argument: sed -i '' "s|...|...|g". Or: brew install gnu-sed && alias sed=gsed
Step 8 — Test end to end CLI
8
Verify the full flow before going live
Grant yourself test credits and walk through the OTP flow.

1. Health check

curl https://prepaired-api.YOURNAME.workers.dev/health
{"status":"ok","version":"1.1.0"}

2. Grant yourself test credits via admin endpoint

curl -X POST https://prepaired-api.YOURNAME.workers.dev/admin/gift \
  -H "X-Admin-Secret: YOUR_ADMIN_SECRET" \
  -H "Content-Type: application/json" \
  -d '{"email": "your@email.com", "credits": 3, "note": "test"}'
{"ok":true,"gifted":1,"results":[{"email":"your@email.com","before":0,"after":3,"added":3}]}

3. Request an OTP

curl -X POST https://prepaired-api.YOURNAME.workers.dev/auth/request-otp \
  -H "Content-Type: application/json" \
  -d '{"email": "your@email.com"}'
{"ok":true,"message":"Code sent to your@email.com. Valid for 10 minutes."}

4. Verify the code (from your inbox)

curl -X POST https://prepaired-api.YOURNAME.workers.dev/auth/verify \
  -H "Content-Type: application/json" \
  -d '{"email": "your@email.com", "code": "123456"}'
{"ok":true,"token":"...","credits":3,"email":"your@email.com"}

5. Open the site and test the Credits tab

Click Get PrepAIred → Use Credits → enter your email → receive code → enter code → Systems Go panel appears → begin interview.

Gifting credits to friends CLI

One curl command. Works for single or bulk. The note field is stored in KV so you can track where credits came from.

Single person

curl -X POST https://prepaired-api.YOURNAME.workers.dev/admin/gift \
  -H "X-Admin-Secret: YOUR_ADMIN_SECRET" \
  -H "Content-Type: application/json" \
  -d '{"email": "friend@example.com", "credits": 5, "note": "beta access"}'

Multiple people at once (up to 100)

curl -X POST https://prepaired-api.YOURNAME.workers.dev/admin/gift \
  -H "X-Admin-Secret: YOUR_ADMIN_SECRET" \
  -H "Content-Type: application/json" \
  -d '{
    "emails": ["alice@ex.com", "bob@ex.com", "charlie@ex.com"],
    "credits": 3,
    "note": "launch cohort"
  }'
{"ok":true,"gifted":3,"results":[...]}

Check anyone's balance

Gift recipients go through the same OTP flow as paying customers — they just have credits already loaded. They enter their email, get a code, verify it, and begin. No special path needed.

Optional — Custom domain Browser + CLI

Point api.prepaired.ijneb.dev at the Worker instead of using the workers.dev URL.

Cloudflare Dashboard → Workers & Pages → prepaired-api → Settings → Domains → Add Custom Domain → enter api.prepaired.ijneb.dev

Then update wrangler.toml and redeploy:

npx wrangler deploy --env production

Then re-run the index.html patch in Step 7 with https://api.prepaired.ijneb.dev as the Worker URL.
